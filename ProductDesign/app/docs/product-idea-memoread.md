# MemoRead 产品想法（草案）

## 1. 场景

我每天上下班通勤时，会在手机上浏览微信公众号、小红书等内容。  
遇到感兴趣的信息时，我现在的做法是先通过企业微信发给自己，等到公司后再看。

这个方式有三个明显问题：

1. 企业微信里的记录难搜索。
2. 无法统计一年内记录过什么。
3. 看不出自己的长期兴趣和学习轨迹。

我希望做一款 SwiftUI 应用，同时支持 iPhone 和 Mac：

1. 通勤路上随手记录。
2. 到公司后在 Mac 上高效整理、回顾、检索。

## 2. 要解决的问题

如何设计一个足够轻量、但可长期积累价值的“灵感记录 + 智能整理”应用。

核心目标：

1. 记录动作足够快（低摩擦）。
2. 回顾动作足够强（可检索、可召回）。
3. 系统能逐步理解我的兴趣（可追踪、可洞察）。

## 3. 产品价值定位

MemoRead 不是“笔记本”，而是“碎片输入的长期收藏夹”：

1. 把零碎时间看到的内容沉淀下来。
2. 把“看过”变成“可回忆、可复用”。
3. 用时间维度还原个人兴趣演化。

## 4. 核心流程

### 4.1 首页

进入应用后展示一个列表（按时间倒序），内容包括：

1. 标题或摘要（自动生成）。
2. 类型标记（链接 / 短文本 / 长文本 / 图片）。
3. 来源与记录时间。
4. 标签或主题。

### 4.2 添加内容

首页提供一个明显的 `添加` 按钮，支持两种入口：

1. 主动添加：用户手动粘贴或输入内容。
2. 自动读取剪贴板：用户开启开关后，每次打开应用自动检测新剪贴板内容，并提示是否保存。

### 4.3 内容类型与 AI 交互

1. 短文本：AI 先识别“关键词记录”还是“想法扩展”，再给出快速选项。
2. 长文本：AI 提示是否生成摘要、关键点、可行动项。
3. 链接：自动抓取标题、域名、摘要（可选）。
4. 图片：OCR 提取文本并生成主题标签（可选）。

### 4.4 后续使用

1. 用户可通过模糊关键词检索历史记录。
2. 系统支持按主题、来源、时间筛选。
3. 定期生成“近期关注主题”与“年度兴趣回顾”。

## 5. 优雅体验的关键设计点

1. 默认轻打扰：自动读取剪贴板只提示一次，不频繁打断。
2. 渐进增强：先完成保存，再提供 AI 处理，不阻塞记录动作。
3. 一致性体验：iPhone 负责快速输入，Mac 负责深度整理，信息结构保持一致。
4. 可解释智能：每个 AI 结果都可编辑、可撤销、可关闭。
5. 强检索优先：搜索和召回体验优先级高于花哨功能。

## 6. MVP（第一版）建议

第一版只做“必须可用”的闭环：

1. 记录列表（新增、编辑、删除、类型显示）。
2. 手动添加 + 剪贴板自动检测保存。
3. 基础搜索（关键词 + 时间）。
4. 基础 AI：短文本关键词提取、长文本摘要。
5. iPhone 和 Mac 同步（本地 + iCloud）。

## 7. 未来可扩展方向

1. 分享扩展（从微信/小红书/浏览器直接保存）。
2. 兴趣图谱（自动聚类长期主题）。
3. 智能回顾（每周/每月自动生成 digest）。
4. 召回问答（“我之前看过关于 xx 的那篇内容是什么？”）。
